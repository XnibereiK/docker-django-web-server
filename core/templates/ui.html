User Interface

Through our UI, we wish to express our focus on the hassle-free experience for the user. Since our system will manage a large amount and variety of data, our UI will have to consist of different views. The current design ideas are presented in the sketches below.

User Interface - Customer View

Venues often have hundreds of people they have to manage and even more they need to serve in order to facilitate easy data capture and its seamless integration to the current workflows, we're going to provide the customers with only 2 main views after login - dashboard and event editing

User Interface - End User View

In order to make the experience of going to the event as pleasant as possible for VIP customers, the bracelet will allow access to certain features through the website, but our goal is to implement features without necessitating a need for display (of course after registration). Website view would only really be used for editing preferences and utilizing the indoor navigation function in 5G enabled locations.
