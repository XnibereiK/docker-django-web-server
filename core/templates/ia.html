Information Architecture

We're delivering a real time location tracking system alongside a touch payment system and ID verification/ ticket verification all for the event organizer to have the ability to view data about the event in real time. This puts a significant expectations on the database setup for this project, which will be the backbone for our information architecture. The frontend has to be an extension of this backbone, and we're putting emphasis on readability and intuitiveness of user experience with this architecture.

Commentary

What basic information does a typical user need to have access to on first interaction with your product?

The first interaction with the product is the landing page. This is a place where they will be able to read and see what the product is and based on this description they should be able to know what role they can register for.

Afterwards, each category of website users will be able to view their respective dashboards. On the dashboard page, they will see the actions they can take and have basic information they filled in registering for the website alongside relevant statistics regarding their data.

What IA Design Principles (Dan Brown) are important for your product?

  1. Principle of Focused Navigation

    The value of our system lies in being able to present complex data to the user in an appoachable and understandable manner. This is why focused navigation is crucial.

  2. Principle of Choices

    While landing page is something that all users will have access to, the rest of the information that can be useful together with the actions that can be executed will only be visible to the intended user. This way noone has access to actions they should not take, but also they will only be exposed to relevant information, which will greatly reduce the headache of figuring out what they can and what they can't do.

  3. Principle of Objects

    Each event is treated like an object that has its own lifecycle and each of them is unique, while following the same, consistent structure. List of past events will be implemented to allow all kinds of users to go back to a specific event for reference if needed.

Which structural Organizational Principles make the most the sense for your product?

  1. Chronological Model

    On the higher level of informational hierarchy, we'll have a list of events, which will be sorted based on the date they have been hosted. This is already implemented in other event management sites such as Eventbrite

  2. Matrix Model

    Ideally, the list of sections would appear to the event organizer in a matrix fashion, where they could choose the suites from the grid that organizes them according to the order they put them in.

  3. Topical Model

    When it comes to the statistics about the event being displayed on the dashboard, they will be sorted based on the topic to make them more organized and user-friendly.

Commentary on Principles of Visual Hierarchy

Visual Hierarchy can enhance user experience when it comes to making things intuitive and obvious without the need to read anything

Principles Used

  1. Color and Contrast

    Contrast can be seen on every page of this website. It makes anything that is important pop out.

  2. Fonts

    Fonts guide the design. The smaller the font, the more specific the text. Title is the largest, then come headers for sections and headers for subsections. Lastly, we have normal paragraph text.

  3. Space

    When presenting the most important information about the product, we want the user to think only about this, no matter what happens next. This is why on the landing page, we can see space after the product intro and tagline.
